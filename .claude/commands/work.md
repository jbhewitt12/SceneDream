# /work command

This is a command to work on an issue. This issue will be complex. You will need to think ultra hard.

1. Look for the issue file in the `/issues` folder that starts with this number: {prompt}
2. Read the issue file and understand the overall problem and the solution.
3. Look at the Completion Notes section to see if any Phases have already been completed.
4. Work on the first Phase that has not yet been completed.
5. If phases are broken up into subsections (eg. Phase 1A, Phase 1B, etc.), work on the first subsection that has not yet been completed.
6. Once you have completed the Phase/subsection, update the Completion Notes section of the issue file with the current status of what you worked on.

**CRITICAL RULE**: ONLY COMPLETE ONE PHASE. Do not complete multiple phases at once.
