# /work command

This is a command to work on an issue. This issue will be complex. You will need to think carefully through the solution.

1. Look for the issue file in the `/issues` folder that starts with this number: $ARGUMENTS
2. Read the issue file and understand the overall problem and the solution.
3. Look at the Completion Notes section to see if any Phases have already been completed.
4. Work on the next phases that have not yet been completed.
5. Once you have completed the issue, update the Completion Notes section of the issue file with the current status of what you worked on.
